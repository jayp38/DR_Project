Group 82 Project Proposal

Introduction/Background

Patients with diabetes may develop diabetic retinopathy (DR), where retinal blood vessels are damaged, potentially causing permanent blindness. Retinal fundus images are scored into five order stages for screening: 0-4 (No DR, Mild, Moderate, Severe, Proliferative). Manual scoring is slow, expensive and inconsistently available, especially in low-resource settings[2].

Automated DR detection has evolved from hand-crafted features to deep learning[2]. Today, transfer learning with CNNs like ResNet provides human-level accuracy[1]. Current work pushes efficiency with models such as EfficientNet and explores next-generation architectures like Vision Transformers[3].

Dataset

Dataset Link: https://www.kaggle.com/competitions/aptos2019-blindness-detection/data

We plan to use APTOS 2019 Blindness Detection Dataset, which contains over 3,500 high-resolution retinal fundus images. Each image was assessed by a clinician and assigned an integer label from 0 to 4, relative to the aforementioned five stages of DR severity. It is representative of real-world data given that there is a significant class imbalance (significantly more healthy images than diseased ones) and a wide range of quality in the images (variations in lighting, focus, and artifacts)[3].

Problem Definition

Diabetic Retinopathy is one of the leading causes of vision loss in adults between the ages of 18 and 65 worldwide, but fortunately, it is largely preventable if and only if detected early[1]. The current way patients are diagnosed is by a trained ophthalmologist manually examining retinal images, a process that creates a significant bottleneck. As such, DR screening is restricted by inefficiency and lack of access. Several patients are unable to attend annual exams due to cost or a shortage of ophthalmologists, which can impede diagnosis and increase cases of preventable blindness[2].

Motivation

We want to build a cost-effective, reliable, and scalable ML pipeline for early DR triage that can:

  • Increase Accessibility: The system can be deployed in pharmacies, clinics, or mobile health units, enabling patient screening without the need of specialists.
  • Improve Speed: The system can analyze an image and provide a risk assessment, allowing for immediate feedback and referral if needed.
  • Boost Efficiency: Since the system can automatically identify healthy eyes or flag high-risk cases, it allows ophthalmologists to focus their limited time and expertise on patients who need them most[1].

Methods

Our approach is going to be centered around supervised learning via Convolutional Neural Networks (CNNs). We will evaluate multiple models that can be used after putting our data through a standardized preprocessing pipeline.

Data Preprocessing

We plan to follow the following procedure:

  • Standardization: All the data (images) will be resized uniformly, and all pixel values will be normalized (0, 1) to make sure that the training is done properly.
  • Data Augmentation: Data will experience multiple transformations (rotations, flips, grayscale, etc.) to prevent model overfitting and improve model generalization.
  • Data/Image Enhancement (TBD): Apply circular crop to the images for black border removal + Contrast Limited Adaptive Histogram Equalization (CLAHE) applied for augmented visibility of features such as microaneurysms and hemorrhages[3].

Modeling Approach

We are planning to compare a series of models to evaluate which works best for this specific use case:

  • Custom CNN: We plan to build a base CNN using TensorFlow and Keras, and this model will likely have 3-4 convolutional layers. This will serve as the benchmark based on which the other modeling techniques will be compared to.
  • Transfer Learning with ResNet50: We plan to fine-tune a pre-trained ResNet50 model (ImageNet), enabling us to replace the final layer and train it on our dataset. This could prove beneficial since we can leverage the pre-trained feature extraction (essential for medical imaging)[1].
  • Advanced Transfer Learning with EfficientNet: Given that EfficientNet is a more modern architecture than ResNet, we plan to fine-tune an EfficientNet (TBD between B3 and B4) to test its computational efficiency and accuracy[3].
  • YOLO: We plan to deploy/evaluate out-of-the-box classification models that Ultralytics offer. Given that YOLO models have differing architectures (CNN-based or Transformer-based), we will compare the balance that the different models have between predictive performance and key efficiency metrics.

(Potential) Results and Discussion

There are a number of metrics that we are going to be using:

  • Cohen's Kappa Score: Will be our primary metric for the overall model performance given that it measures agreement beyond chance (makes it ideal for multi-class medical diagnosis)[1].
  • Confusion Matrix: Visualization to analyze model's specific error patterns and will highlight where model improvement is needed.
  • Precision, Recall, F1-Score: Will record performance per class, priority will be placed on recall to minimize false negatives (most dangerous in this context) and macro-averaged F1-score to ensure multi-class robust performance[2].
  • Balanced Accuracy Score: Calculated as average recall per class, meaning that it prevents high accuracy from hiding poor performance on critical/severe classes (important because raw accuracy could be high but model might miss all positive diagnoses).
